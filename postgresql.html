🟢 Table Partitioning (Jadval Bo'lish)
🎯 Nima uchun kerak? Juda katta jadvallarni (millionlab qator) kichik bo'laklarga ajratish. So'rov faqat kerakli bo'lakni ko'radi → tezroq ishlaydi.
-- RANGE partitioning (sana bo'yicha ajratish):
CREATE TABLE orders (id SERIAL, total NUMERIC, created_at DATE)
  PARTITION BY RANGE (created_at);                          ◄--- Sana bo'yicha bo'lish
CREATE TABLE orders_2025 PARTITION OF orders
  FOR VALUES FROM ('2025-01-01') TO ('2026-01-01');           ◄--- 2025 yil
CREATE TABLE orders_2026 PARTITION OF orders
  FOR VALUES FROM ('2026-01-01') TO ('2027-01-01');           ◄--- 2026 yil

-- LIST partitioning (hudud bo'yicha):
CREATE TABLE users (id SERIAL, name TEXT, region TEXT)
  PARTITION BY LIST (region);
CREATE TABLE users_uz PARTITION OF users FOR VALUES IN ('UZ');
CREATE TABLE users_kz PARTITION OF users FOR VALUES IN ('KZ');

🟢 Backup & Restore (Zaxira va Tiklash)
🎯 Nima uchun kerak? Ma'lumotlar yo'qolishidan himoya! Server buzilsa, xaker hujum qilsa — zaxiradan tiklash mumkin. Har kuni avtomatik zaxira olish — eng muhim vazifa!

# Zaxira olish:
pg_dump ecommerce > backup.sql                     ◄--- SQL format (o'qish mumkin)
pg_dump -Fc ecommerce > backup.dump                ◄--- Siqilgan format (tezroq tiklash)
pg_dump --schema-only ecommerce > schema.sql       ◄--- Faqat tuzilish (ma'lumotsiz)
pg_dump --data-only ecommerce > data.sql           ◄--- Faqat ma'lumot (tuzilishsiz)

# Tiklash:
psql ecommerce < backup.sql                        ◄--- SQL dan tiklash

pg_restore --clean -d ecommerce backup.dump        ◄--- Avval tozalab, keyin tiklash
